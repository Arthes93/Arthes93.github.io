추상화와 구체화 둘다 의존하는 문제가 있다.
=> DIP 위반

DIP : 구현 class가 아닌 추상화 되어 있는 Interface에 의존해야 한다.
DIP의 원칙을 지키기 위해 의존 class내에서 구현 클래스를 가져오는 부분을 삭제
객체가 존재하지 않아서 nullPointException이 발생한다.
클래스 외부에서 누군가가 직접 객체를 생성해 줘야 한다.
